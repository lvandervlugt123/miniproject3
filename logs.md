
Project 3 – Learning Log (Pokemon Team Builder API Project)
Iteration 1 – Basic API Integration & Display
In my first iteration, I focused on building a simple application that fetched and displayed Pokémon data from an API (PokeAPI). The goal was to understand how API requests work using JavaScript, specifically using fetch() to retrieve data and render it dynamically on the page.
One challenge I faced was handling asynchronous data and making sure the Pokémon information appeared correctly after the API response loaded. I learned how to use promises and structure my code so that the DOM only updates once the data is available. This iteration helped me build a strong foundation in API integration and dynamic content rendering.

Iteration 2 – Team Creation & User Interaction
In the second iteration, I added functionality that allowed users to create their own Pokémon team, starting with a selected starter Pokémon. Users could then add additional Pokémon to build a full team.
This required implementing event listeners and managing user input, which deepened my understanding of DOM manipulation and state management. One key challenge was tracking which Pokémon were already selected and updating the interface accordingly.

Iteration 3 – Advanced Features (Battling & Evolutions)
In my final iteration, I expanded the application by adding more complex features, including the ability to simulate battles between teams and view Pokémon evolutions.
The battle feature required conditional logic and comparison between teams, which improved my problem-solving skills in JavaScript.

Design & User Experience
Throughout the project, I focused on designing the app with a Pokémon theme in mind, making it visually engaging and intuitive. I learned how design choices impact usability, especially when dealing with multiple interactive elements like team selection and battle features.
I also improved my ability to balance functionality and design, ensuring the app was both useful and enjoyable to use.
